# Kibana API Reference

API operations for creating observability resources on a running Kibana instance.
All endpoints require the `kbn-xsrf: true` header for CSRF protection.

## Reading connection details

When the developer asks you to make API calls, auto-detect the connection details
from the config files — don't ask the developer to look these up:

```bash
# Kibana port from kibana.dev.yml
kibana_port=$(grep -E "^ *port:" config/kibana.dev.yml | head -1 | awk '{print $2}')
kibana_url="http://localhost:${kibana_port}"

# ES credentials (for authenticated endpoints)
es_user=$(grep -E "^ *username:" config/kibana.dev.yml | head -1 | awk '{print $2}' | tr -d '"')
es_pass=$(grep -E "^ *password:" config/kibana.dev.yml | head -1 | awk '{print $2}' | tr -d '"')
```

Default credentials for local ES: `elastic` / `changeme`

Read these values programmatically before constructing any curl commands.

## Wait for Kibana readiness

Always verify Kibana is ready before making API calls:

```bash
until curl -s "http://localhost:${kibana_port}/api/status" | grep -q '"overall":{"level":"available"'; do
  sleep 5
done
```

## Create SLO

**Prerequisites:** Before creating an SLO, data must exist in the target index.
Run `run-data slo` first to ingest fake_stack data into `high-volume-metrics`.
Auto-detect the Kibana repo directory from the worktree config — don't tell the
developer to navigate anywhere manually.

**Endpoint:** `POST /api/observability/slos`

**Headers:**
- `kbn-xsrf: true`
- `Content-Type: application/json`

**Required fields:**
- `name` — SLO display name
- `budgetingMethod` — `"occurrences"` or `"timeslices"`
- `description` — human-readable description
- `indicator` — metric definition (type + params)
- `objective` — target and time window

### Example: Admin Console Availability SLO (using kbn-data-forge fake_stack data)

This is the standard SLO type used in development. It measures availability based on
HTTP status codes, not latency. The data is generated by `run-data slo` (which uses
`kbn-data-forge` with the `fake_stack` dataset) and lands in `high-volume-metrics`
with fields like `http.response.status_code`, `service.name`, etc.

```bash
curl -X POST "http://localhost:${kibana_port}/api/observability/slos" \
  -H "kbn-xsrf: true" \
  -H "Content-Type: application/json" \
  -u "${es_user}:${es_pass}" \
  -d '{
    "name": "Admin Console Availability SLO",
    "description": "99% of requests return non-5xx responses",
    "budgetingMethod": "occurrences",
    "indicator": {
      "type": "sli.kql.custom",
      "params": {
        "index": "high-volume-metrics",
        "filter": "",
        "good": "http.response.status_code < 500",
        "total": "http.response.status_code : *"
      }
    },
    "timeWindow": {
      "duration": "30d",
      "type": "rolling"
    },
    "objective": {
      "target": 0.99
    }
  }'
```

### Example: APM Latency SLO

```bash
curl -X POST "http://localhost:${kibana_port}/api/observability/slos" \
  -H "kbn-xsrf: true" \
  -H "Content-Type: application/json" \
  -u "elastic:changeme" \
  -d '{
    "name": "Checkout Latency SLO",
    "description": "95% of checkout transactions under 2s",
    "budgetingMethod": "occurrences",
    "indicator": {
      "type": "sli.apm.transactionDuration",
      "params": {
        "service": "checkout-service",
        "environment": "production",
        "transactionType": "request",
        "transactionName": "POST /api/checkout",
        "threshold": 2000,
        "index": "metrics-apm*"
      }
    },
    "timeWindow": {
      "duration": "30d",
      "type": "rolling"
    },
    "objective": {
      "target": 0.95
    }
  }'
```

### List SLOs

```bash
curl -s "http://localhost:${kibana_port}/api/observability/slos" \
  -H "kbn-xsrf: true" \
  -u "elastic:changeme" | jq '.results[] | {id, name, status}'
```

### Delete SLO

```bash
curl -X DELETE "http://localhost:${kibana_port}/api/observability/slos/<slo-id>" \
  -H "kbn-xsrf: true" \
  -u "elastic:changeme"
```

## Create Synthetics Private Location

Private locations require a Fleet agent policy. The `kibana.dev.yml` template already
provisions one via `xpack.fleet.agentPolicies` with ID `fleet-server-policy` — so you
do NOT need to create a policy manually. Just create the private location pointing to it.

**Endpoint:** `POST /api/synthetics/private_locations`

**Headers:**
- `kbn-xsrf: true`
- `Content-Type: application/json`

**Required fields:**
- `label` — location display name
- `agentPolicyId` — Fleet agent policy ID (use `fleet-server-policy` from kibana.dev.yml)

**Optional fields:**
- `geo` — `{ "lat": number, "lon": number }` (WGS84 coordinates)
- `tags` — string array
- `spaces` — string array of Kibana space IDs (defaults to all)

### Create private location

```bash
curl -X POST "http://localhost:${kibana_port}/api/synthetics/private_locations" \
  -H "kbn-xsrf: true" \
  -H "Content-Type: application/json" \
  -u "elastic:changeme" \
  -d '{
    "label": "Local Dev Private Location",
    "agentPolicyId": "fleet-server-policy",
    "geo": {
      "lat": 41.12,
      "lon": -71.34
    }
  }'
```

### List private locations

```bash
curl -s "http://localhost:${kibana_port}/api/synthetics/private_locations" \
  -H "kbn-xsrf: true" \
  -u "elastic:changeme" | jq '.[]'
```

### Delete private location

```bash
curl -X DELETE "http://localhost:${kibana_port}/api/synthetics/private_locations/<location-id>" \
  -H "kbn-xsrf: true" \
  -u "elastic:changeme"
```

## Spaces

Useful for testing features that are space-aware (e.g., private locations scoped to spaces).

### Create space

```bash
curl -X POST "http://localhost:${kibana_port}/api/spaces/space" \
  -H "kbn-xsrf: true" \
  -H "Content-Type: application/json" \
  -u "elastic:changeme" \
  -d '{
    "id": "test-space",
    "name": "Test Space",
    "description": "Space for testing"
  }'
```

### List spaces

```bash
curl -s "http://localhost:${kibana_port}/api/spaces/space" \
  -H "kbn-xsrf: true" \
  -u "elastic:changeme" | jq '.[].id'
```

### API calls in a specific space

Prefix the endpoint with `/s/<space-id>`:

```bash
curl -X POST "http://localhost:${kibana_port}/s/test-space/api/observability/slos" \
  -H "kbn-xsrf: true" \
  -H "Content-Type: application/json" \
  -u "elastic:changeme" \
  -d '{ ... }'
```
